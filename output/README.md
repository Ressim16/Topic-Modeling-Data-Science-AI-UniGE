# Output Directory

This directory contains all outputs generated during the topic modeling analysis. Due to file size and professionaly confidential constraints, most output files are **not tracked in Git** (see `.gitignore`).

## Directory Structure

```
output/
├── vectors/          # Embedding vectors and transformed representations
├── figures/          # Visualizations and plots
├── llm/              # LLM-generated topic labels and descriptions
└── modeling/         # Saved models and clustering results
```

---

## Subdirectories

### `vectors/`
**Contains**: Document embeddings and dimensionality-reduced representations

- `embeddings.npy` - BioBERT embeddings for all documents
- `umap_embeddings.npy` - UMAP-reduced embeddings (2D/3D)
- Document-term matrices
- Feature vectors

**File types**: `.npy`, `.pkl`

**Size**: ~100MB - 1GB depending on dataset size

### `figures/`
**Contains**: All visualizations generated during analysis

Expected files:
- `publications_over_time.png` - Temporal trend analysis
- `journal_distribution.png` - Top journals bar chart
- `topic_distribution.png` - Topic prevalence across corpus
- `topic_map_2d.png` - 2D visualization of topics
- `topic_similarity_heatmap.png` - Topic relationship matrix
- `cluster_quality_metrics.png` - Silhouette scores, etc.

**File types**: `.png`, `.pdf`, `.svg`, `.html` (for interactive plots)

**Usage**: Include these in presentations, reports, or papers

### `llm/`
**Contains**: LLM-generated topic refinements

Expected files:
- `topic_labels.json` - Structured topic descriptions
- `topic_summaries.json` - Detailed topic interpretations
- `refinement_log.txt` - LLM processing log

**Format**: JSON with structure:
```json
{
  "topic_0": {
    "label": "Reproductive Health and Fertility",
    "description": "Research on dairy cattle reproduction...",
    "keywords": ["fertility", "reproduction", "breeding"]
  }
}
```

### `modeling/`
**Contains**: Saved models and clustering results

Expected files:
- `cluster_assignments.csv` - Document-to-topic mappings
- `topic_summary.csv` - Topic statistics and metadata
- `hdbscan_labels.npy` - Clustering labels
- `umap_coords_full.npy` - Dimensionality reduction output
- `vectorizer_model.pkl` - Vectorizer model

**File types**: `.pkl`, `.npy`, `.csv`

---

## Generating Outputs

All outputs are generated by running `Rapport_Unige_Data_Science.ipynb`. 

### To regenerate outputs:

1. Ensure you have the required data in `data/` directory
2. Open the Jupyter notebook
3. Run all cells sequentially
4. Outputs will be automatically saved to respective subdirectories

---

## File Size Considerations

Most output files are excluded from Git due to size:
- Embedding vectors can be 500MB+
- Saved models can be 1GB+
- Figure files are typically small (<10MB total)

---

## Output Versioning

When making significant changes to the analysis:
1. Create a new output subdirectory with timestamp: `output_YYYYMMDD/`
2. Update paths in notebook accordingly
3. Document changes in commit message
